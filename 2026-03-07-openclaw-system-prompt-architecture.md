# OpenClaw System Prompt 9层架构详解

**时间戳**: 2026-03-07
**来源**: @servasyy_ai (黄宗宁) - https://x.com/servasyy_ai/status/2029489020208848966
**版本**: v2.1

## 核心架构

OpenClaw Agent System Prompt 不是一个单一文件，而是 **9 层架构** 的精心编排。

### 框架层 (Layer 1-6 + 9) - 自动生成
- Layer 1: Framework Core - 框架核心层
- Layer 2: Tool Definitions - 工具定义层
- Layer 3: Skills Registry - 技能注册表
- Layer 4: Model Aliases - 模型别名层
- Layer 5: Protocol Specifications - 协议规范层
- Layer 6: Runtime Info - 运行时信息层
- Layer 9: Inbound Context - 入站上下文层

### 用户可控层 (Layer 7 + 8)
- Layer 7: Workspace Files - 工作区文件层（静态可编辑）
- Layer 8: Bootstrap Hook System - 动态注入层（可编程）

## 可借鉴的架构设计原则

### 1. 分层清晰，职责明确
- **框架层**: 保证一致性和稳定性，所有 Agent 自动获得新能力
- **用户层**: 提供灵活性和个性化，支持版本管理和共享
- **分离关注点**: 静态配置与动态注入分离，框架层与用户层分离

### 2. 用户可控机制设计

#### Layer 7（Workspace Files）- 静态配置
- 用户可以直接编辑的配置文件
- 适用场景: 定义 Agent 身份、工作规范、记忆
- 优点: 简单直观，易于版本管理
- 核心文件:
  - IDENTITY.md - Agent 身份定义
  - AGENTS.md - 团队协调规范
  - MEMORY.md - 持久化记忆
  - SOUL.md - 核心身份和原则
  - USER.md - 用户档案

#### Layer 8（Bootstrap Hook System）- 动态注入
- 四种 Hook 机制:
  1. **agent:bootstrap Hook** - 完全控制 bootstrapFiles 数组
  2. **bootstrap-extra-files Hook** - 只追加文件，不修改现有文件
  3. **before_prompt_build Hook** - 修改最终 prompt（在发送给 LLM 前）
  4. **bootstrapMaxChars / bootstrapTotalMaxChars** - 控制字符预算

### 3. 设计权衡的透明化
每层都明确说明设计决策、权衡（Tradeoff）、好处和代价，例如:
- **灵活性 vs 一致性**: 框架层统一生成，用户层允许个性化
- **Token 消耗 vs 上下文准确性**: 每次请求都注入最新运行时状态
- **静态配置的简单性 vs 动态注入的灵活性**

### 4. 自动化与规范化
- **目录扫描**: 自动扫描 skills 目录，添加新 Skill 只需放入目录
- **JSON Schema**: 使用严格类型定义工具参数，支持自动验证和文档生成
- **标准化协议**: 定义 Silent Replies、Heartbeats、Reply Tags 等交互协议

### 5. 比喻驱动理解
每层都使用生动的比喻帮助理解:
- Layer 1 - "使用说明"
- Layer 2 - "瑞士军刀工具清单"
- Layer 3 - "餐厅特色菜谱"
- Layer 6 - "仪表盘"
- Layer 9 - "实时路况信息"

### 6. 优化策略

#### 静态文件优化（Layer 7）
- 保留核心框架，删除冗余描述
- 使用表格代替段落
- 依赖 MemOS 自动导出，不手动添加内容
- 避免重复框架已知的事情

#### Hook 系统优化（Layer 8）
- 简单场景: 使用 bootstrap-extra-files
- 复杂场景: 使用 agent:bootstrap 支持条件判断
- 动态场景: 使用 before_prompt_build 注入实时上下文
- 避免: 耗时操作、过多内容、不稳定外部依赖

### 7. 快速导航设计（TL;DR）
- 新手必读: 突出最常用的两层（Layer 7 和 8）
- 常见需求: 提供场景到解决方案的快速映射
- 省略细节: 其他层了解即可

### 8. 大小对比和 Token 优化
- 明确标注各层大小估算值
- 说明框架层理论上应该相同但可能因配置差异
- 提供提示词裁剪策略（头 70% + 尾 20%）

## 启示

### 对 AI 系统设计
1. **分层架构是复杂系统的基础**: 将框架、工具、配置、运行时上下文分离
2. **用户可控性设计**: 提供多个层次的定制能力，从静态文件到动态脚本
3. **自动化 vs 灵活性**: 在自动化的同时保留足够的灵活性供用户定制
4. **设计决策透明化**: 明确说明每个决策的权衡，帮助用户理解系统设计

### 对文档设计
1. **比喻驱动理解**: 使用生动的比喻降低学习门槛
2. **快速导航**: 提供 TL;DR 和场景映射，满足不同深度的需求
3. **权衡透明化**: 明确说明设计决策的好处和代价
4. **实用主义**: 每层都说明"为什么这样设计"和"如何使用"

### 对团队协作
1. **标准化协议**: 定义统一的交互协议，便于多 Agent 协作
2. **版本管理和共享**: 用户配置文件可以版本管理、备份、共享
3. **自动化降低风险**: 通过目录扫描、类型验证等降低配置错误风险

## 适用场景

这种架构设计特别适合:
- 复杂的 AI Agent 系统
- 需要高度定制化的 AI 应用
- 多 Agent 协作场景
- 需要版本管理和团队协作的项目

## 核心价值

OpenClaw 的 System Prompt 架构设计的核心价值在于:
1. **平衡**: 在一致性和灵活性之间找到最佳平衡点
2. **分层**: 通过清晰的分层降低复杂度
3. **自动化**: 通过自动化降低配置错误风险
4. **用户可控**: 提供多层次的用户可控机制
5. **透明**: 设计决策透明化，降低理解成本
